devops-hub

O cérebro central de CI/CD e governança da org — e o pipeline que transforma uma especificação em código testado, com PR e CI, parando sozinho quando o risco é clínico.

github.com/lifesystemsufpr/devops-hub

01A ideia

Por que isso existe.

Hub-and-spoke

Toda a lógica de CI (lint, typecheck, testes, build, cobertura) vive num lugar só. Cada repo só consome um ci.yml de ~15 linhas. Atualizar o CI dos 7 repos = um commit no hub.

Automação por risco

Rotineiro = automatizado ponta a ponta. Clínico, autenticação e schema = revisão humana obrigatória. Software de saúde não admite full-autopilot.

05Como vira produção

O seam é único e claro.

  • O cérebro: trocar o mapa CANNED_IMPLS em pipeline/generator.ts por uma chamada ao Claude Agent SDK ou Cursor SDK, passando o spec + as regras do ai-toolkit como contexto.
  • O gatilho: deixa de ser npm run pipeline e passa a ser uma task do ClickUp (integração nativa ClickUp↔Cursor) ou uma issue (Copilot coding agent).
  • O resto não muda: validação, PR e CI continuam idênticos. O guard-rail vem das regras commitadas; o CODEOWNERS força reviewer em área de risco.
